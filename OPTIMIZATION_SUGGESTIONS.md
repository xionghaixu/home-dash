# home-dash 后端阶段规划与交付清单

> 文档定位：
> - 本文档仅维护阶段目标、交付边界、功能清单、验收口径。
> - 编码规范、异常规范、日志规范请查看 `SKILLS_SPEC.md`。

## 1. 项目主线

`home-dash` 后端主线：

1. 阶段一：核心网盘可用版（已完成）
2. 阶段二：预览与检索整理
3. 阶段三：媒体中心与 NAS 体验
4. 阶段四：数据治理与任务中心
5. 阶段五：家庭设备接入与备份
6. 阶段六：登录与安全体系

## 3. 阶段一标准能力（已完全完成）

### 3.1 标准功能清单

- 文件列表查询、目录切换、面包屑路径
- 文件/文件夹创建、重命名、删除、移动、复制
- 单文件上传、分块上传、断点续传、分块合并
- MD5 预检、秒传、文件完整性校验、分块完整性校验
- 传输任务列表、状态清理、单条清理
- 系统信息统计接口
- 视频播放基础资源支撑
- 统一响应模型 `ResponseDto`
- 全局异常处理、业务异常体系、错误码体系

### 3.2 阶段一验收结论

- 阶段一功能链路闭环完整，可独立部署使用
- 阶段一不再新增功能，所有新增能力统一进入后续阶段

## 4. 阶段二到阶段六的前后端对齐原则

- 后端新增一个业务模块，前端只增加一组必要页面或复用现有抽屉/弹层，不做重复入口。
- 详情类能力优先通过统一详情结构输出，前端优先复用详情抽屉，不单独拆多个详情页。
- 预览类能力优先使用弹层、抽屉、底部播放条等轻量承载，不为单一文件类型新增过多独立页面。
- 长耗时能力统一任务化，前端统一在任务中心或当前页面状态区感知，不为每类任务单独造一套状态系统。
- 每阶段必须做到：后端功能模块、接口输出、前端页面入口、交互闭环四者同步定义。

## 5. 阶段二：预览与检索整理

### 5.1 阶段目标

解决“快速找到文件、看懂文件、进行轻量整理”的核心问题。

### 5.2 后端功能模块

#### A. 搜索检索模块

- 文件名关键词搜索
- 标签关键词搜索
- 组合筛选：类型、时间范围、大小区间、目录范围、是否收藏
- 统一分页、排序、结果总数统计
- 搜索历史记录与热门筛选项统计
- 常用筛选视图保存与读取

#### B. 详情增强模块

- 文件详情增强字段：扩展名、大小、类型、父目录、路径、资源标识、更新时间
- 输出关联信息：同目录文件、同类型文件、最近上传关联项
- 输出预览能力标记：是否支持预览、预览类型、预览状态、预览失败原因
- 输出整理状态：是否收藏、标签列表、最近访问时间

#### C. 图片预览模块

- 原图访问接口
- 缩略图访问接口
- 图片尺寸信息
- 图片拍摄时间、拍摄设备等基础 EXIF 字段
- 预览缓存状态与生成时间字段

#### D. 文本预览模块

- 文本文件安全读取接口
- 大文本分段读取与最大读取限制
- 编码识别结果输出
- 代码/日志类文本等宽预览支持字段
- 文本摘要与关键片段输出

#### E. 音频预览模块

- 音频播放地址接口
- 音频时长、标题、专辑、歌手等元数据输出
- 封面字段预留
- 播放失败降级说明
- 最近播放记录写入接口

#### F. 轻量整理模块

- 收藏标记与取消
- 最近使用记录
- 标签增删改查
- 批量标签与批量收藏
- 文件备注或短描述字段预留

### 5.3 数据与接口交付

- 新增搜索与筛选请求 DTO、分页响应 VO
- 新增统一文件详情扩展 VO
- 新增预览资源与预览状态字段
- 新增收藏、标签、最近使用、搜索历史基础表结构
- 新增批量整理接口与批量结果回执结构

### 5.4 页面收敛要求（与前端一致）

- 搜索与筛选统一收敛到一个搜索页，不拆高级搜索页与普通搜索页两套入口
- 文件详情统一由详情抽屉承载，不拆“详情页 + 详情弹窗”双模式
- 图片/文本预览优先弹层，音频优先底部播放条或单页二选一

### 5.5 验收口径

- 支持“搜索 -> 详情 -> 预览 -> 收藏/标签”完整闭环
- 预览失败可感知、可解释、可降级
- 阶段二页面不超过一套搜索主入口、一套详情抽屉、一组预览承载方式

## 6. 阶段三：媒体中心与 NAS 体验

### 6.1 阶段目标

将文件管理升级为内容管理，形成图片、视频、音频三大独立媒体能力，并提供统一媒体入口。

### 6.2 后端功能模块

#### A. 首页聚合模块

- 最近上传摘要
- 最近播放、最近观看摘要
- 图片回顾摘要
- 媒体数量与容量摘要
- 收藏媒体摘要
- 待处理媒体任务摘要

#### B. 图片媒体模块

- 按年、月、日、目录聚合图片
- 时间线数据结构
- 多尺寸缩略图字段
- 相册分组字段（自动相册/手动相册）
- 图片拍摄时间、拍摄地点、拍摄设备字段
- 连续浏览与幻灯片顺序数据

#### C. 视频媒体模块

- 时长、分辨率、封面元数据
- 最近观看与继续播放记录
- 视频合集/系列聚合字段
- 字幕关联结构与字幕列表预留
- 清晰度、码率、封面生成状态字段
- 同系列推荐和相关推荐基础字段

#### D. 音频媒体模块

- 标题、专辑、歌手、时长元数据
- 专辑聚合与歌手聚合
- 最近播放记录
- 播放列表与收藏夹关联
- 曲目序号、年份、封面字段
- 随机播放、顺序播放所需的列表排序数据

#### E. 媒体任务模块

- 媒体元数据提取任务化
- 缩略图、封面生成任务化
- 失败重试与任务状态可查询
- 手动重扫、批量重扫
- 新上传媒体自动入队扫描

### 6.3 数据与接口交付

- 新增首页媒体聚合接口
- 新增图片、视频、音频三类列表接口与详情接口
- 新增播放进度、最近观看、最近播放、相册、播放列表基础表结构
- 新增媒体扫描任务与封面生成任务结果结构

### 6.4 页面收敛要求（与前端一致）

- 首页只保留一个媒体工作台入口，不再单独新增多个摘要页
- 图片、视频、音频各保留一个主列表页，详情优先在抽屉或增强播放页承载
- 相册、播放列表、系列聚合优先作为子视图或面板，不新增过多独立导航项

### 6.5 验收口径

- 图片、视频、音频具备独立查询模型与接口
- 首页可作为媒体内容入口的数据来源
- 媒体浏览能力形成“首页 -> 列表 -> 播放/预览 -> 继续浏览”闭环

## 7. 阶段四：数据治理与任务中心

### 7.1 阶段目标

提供可持续的数据治理能力，支持长期稳定运行。

### 7.2 后端功能模块

#### A. 回收站模块

- 软删除
- 恢复到原目录或指定目录
- 彻底删除与批量清空
- 删除来源、删除时间、原路径记录
- 自动过期清理策略

#### B. 重复文件治理模块

- 基于 MD5/名称/大小分组
- 建议保留项与清理候选项
- 按目录范围、类型范围发起重复检测
- 重复检测任务化
- 批量清理与批量保留结果回写

#### C. 存储分析模块

- 容量占比、类型占比
- 目录占用排行、大文件排行
- 增长趋势分析
- 清理建议摘要
- 目录热点变化与月度增长趋势
- 可清理空间预估

#### D. 统一任务中心模块

- 统一任务模型：状态、进度、来源、错误摘要、开始时间、结束时间
- 任务重试、取消、历史查询
- 任务类型覆盖：扫描、统计、治理、导入、备份
- 支持最近失败任务、最近完成任务快速查询
- 支持任务结果摘要与结果详情查询

#### E. 操作记录与巡检模块

- 关键操作审计记录
- 孤儿资源检测
- 失效预览资源检测与清理记录
- 文件完整性巡检
- 巡检报告与异常摘要

#### F. 大文件与空目录治理模块

- 大文件识别与排序
- 空目录识别
- 长期未访问文件识别
- 一键加入治理任务队列

### 7.3 数据与接口交付

- 新增回收站、重复组、任务、巡检报告、操作日志基础表结构
- 新增统一任务结果模型与分页查询接口
- 新增治理动作回执与撤销能力预留字段

### 7.4 页面收敛要求（与前端一致）

- 治理类功能优先收敛为“治理中心 + 任务中心”两大入口
- 重复文件、大文件、空目录等治理结果优先在同一治理框架内切换，不拆太多平级页面
- 巡检报告与操作记录优先复用统一筛选与表格容器

### 7.5 验收口径

- 治理能力可执行、可追踪、可回看
- 长耗时任务统一纳管，不再散落在日志
- 用户可在较少页面入口内完成主要治理动作

## 8. 阶段五：家庭设备接入与备份

### 8.1 阶段目标

让系统成为家庭设备与存储来源的统一汇聚中枢。

### 8.2 后端功能模块

#### A. 导入与同步模块

- 本地目录导入
- 外部存储导入
- 手机相册导入协议预留
- 导入历史与失败重试
- 同步规则与冲突命名策略
- 冲突记录查询
- 增量扫描与全量扫描模式

#### B. 备份计划模块

- 备份计划（来源、目标、周期）
- 手动立即执行
- 执行历史与失败重试
- 备份结果校验
- 版本化备份预留
- 最近成功时间与最近失败原因摘要

#### C. 恢复与校验模块

- 恢复演练记录
- 单文件恢复与目录恢复预留
- 备份完整性校验
- 校验异常结果记录
- 恢复前检查结果输出

#### D. 设备与来源管理模块

- 设备概览
- 来源状态管理
- 最近导入/备份状态
- 磁盘健康与容量预警字段
- 来源可用性检测
- 设备最近联机时间字段

#### E. 通知与告警模块

- 备份失败告警摘要
- 容量不足告警摘要
- 导入异常告警摘要
- 最近一次告警确认状态

### 8.3 数据与接口交付

- 新增来源、设备、导入任务、同步规则、备份计划、备份结果基础表结构
- 新增告警摘要结构与最近执行状态接口
- 新增导入/备份统一结果查询接口

### 8.4 页面收敛要求（与前端一致）

- 导入、同步、来源管理优先收敛为“来源中心”
- 备份、恢复、校验优先收敛为“备份中心”
- 设备概览与磁盘健康优先作为设备中心子模块，不拆散为多个无强关联页面

### 8.5 验收口径

- 导入、同步、备份形成端到端闭环
- 关键失败可定位、可重试、可补偿
- 页面入口控制在来源中心、备份中心、设备中心三组以内

## 9. 阶段六：登录与安全体系（延后）

### 9.1 阶段目标

完善公网与多用户场景下的访问安全和权限体系。

### 9.2 后端功能模块

#### A. 认证模块

- 登录/退出
- 密码校验与密码修改
- 登录失败次数控制
- 图形验证码或校验码预留
- 找回密码流程预留

#### B. 账号与角色模块

- 账号体系（单用户密码或家庭成员账号）
- 角色模型
- 基础权限分配
- 账号启用/停用状态
- 默认管理员初始化流程

#### C. 会话与可信设备模块

- 会话管理
- 可信设备管理
- 会话失效与强制下线
- 最近登录记录
- 异地登录提醒预留

#### D. 分享与访问控制模块

- 分享链接权限与过期策略
- 提取码机制预留
- 下载次数限制预留
- 分享访问记录
- 分享关闭与撤回能力

#### E. 安全防护模块

- 限流、防爆破、审计
- 操作审计日志
- IP 黑白名单预留
- 风险操作二次确认预留
- 高风险登录标记预留

#### F. 域名与 SSL 模块

- 域名接入配置说明
- SSL 配置说明
- 反向代理接入说明字段
- 公网访问模式标记

### 9.3 数据与接口交付

- 新增账号、角色、权限、会话、可信设备、分享记录、安全审计基础表结构
- 新增登录状态、分享状态、安全事件摘要接口
- 新增权限校验中间层或统一鉴权结构

### 9.4 页面收敛要求（与前端一致）

- 登录单独一页，安全相关能力尽量收敛到账号安全中心
- 权限、会话、可信设备、分享安全优先以标签页或分区呈现，不拆过多独立页面
- 域名与 SSL 以引导说明页或配置区承载，不单独扩展成复杂管理台

### 9.5 验收口径

- 多用户和公网访问场景可稳定运行
- 核心接口具备权限控制与审计闭环
- 前端页面数量受控，安全入口保持集中

## 10. 跨阶段统一约束

- API 前缀统一 `/v1`
- 响应结构统一 `ResponseDto`
- 长耗时能力优先任务化
- 每阶段新增能力必须同步更新“目标 + 功能清单 + 验收口径”
- 每阶段的后端模块数与前端页面/组件规模保持同一节奏，不允许后端规划过多而前端无入口承接

## 11. 实施顺序建议

1. 从阶段二开始，优先搜索与三类预览能力。
2. 阶段三再进入媒体中心，避免提前引入重型治理逻辑。
3. 阶段四统一承接治理与任务中心。
4. 阶段五承接设备与备份联动。
5. 阶段六统一处理登录与安全，不打散前五阶段主线。
